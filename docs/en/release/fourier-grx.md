## Release Notes

### 4.4.25

> 📅 2026-07-16 &nbsp;·&nbsp; Platform: `linux/arm64`

🔧 **Changed**

- **Mark Time Assist DT mode `assist_ratio` decayed too fast**: in `AlgorithmM4LRotaryJointMarkTimeAssistAdjustDtControlModel`, when the measured torque drops below the lower threshold, the `assist_ratio` decay step is reduced from `-0.01` to `-0.001`, avoiding unstable assist behavior caused by the assist ratio falling off too quickly during mark time. Forward Walk Assist DT mode is not affected.

---

### 4.4.24

> 📅 2026-07-16 &nbsp;·&nbsp; Platform: `linux/arm64`

🔧 **Changed**

- **Reference trajectory time-domain alignment (Forward Walk & Mark Time)**: enabled reference trajectory delay compensation (`gait_trajectory_total_delay_compensation_index = -6`), correcting the time-domain offset between the `reference_joint_position` uploaded to the GUI and the robot's actual `measured_joint_position`, improving the visual alignment of the two curves in the GUI

- **Faster Assist DT mode response** (Forward Walk & Mark Time): the `assist_ratio` increment on assist trigger changed from `0.01` to `0.02`, doubling the assist ramp-up speed for a faster response

- **M4LP1 / M4LT1 load mode kp parameter tuning**: `algo_rotary_pd_kp_hard` hip joints (indices 0, 2) changed from `500` to `600`, knee joints (indices 1, 3) unchanged at `500`, improving stiffness during loaded walking

---

### 4.4.23

> 📅 2026-07-06 &nbsp;·&nbsp; Platform: `linux/arm64`

🐛 **Fixed**

- **Forward walk step length was too short**: `hip_movement_ratio=(1.0, 0.7)` scales the backward hip swing angle to 70% after IK, so the actual physical step length was only 85% of the commanded step length (in a symmetric step, the forward swing contributes `S/2×1.0` and the backward swing contributes `S/2×0.7`, totaling `0.85S`). Pre-multiplied by a compensation factor of `20/17` (≈ 1.176) at the Cartesian level so the commanded step length matches the actual step length.

  Covers all of the following forward walk tasks (fixed in one place via the `_run_gait_generation` inheritance chain):
  - TID 4111 Passive Forward Walking
  - TID 4116 Assisted Forward Walking (PD)
  - TID 4118 Assisted Forward Walking (DT)
  - TID 4301 Knee-Restricted Forward Walking
  - TID 4303 Knee-Restricted Assisted Forward Walking (PD)
  - TID 4305 Knee-Restricted Assisted Forward Walking (DT)

  The Mark Time series uses a separate `generate_mark_gait` with no step-length parameter and is not affected.

---

### 4.4.22

> 📅 2026-07-02 &nbsp;·&nbsp; Platform: `linux/arm64`

✨ **Added**

- **Rotary joint manual calibration task** (TID 4121, `TASK_ROTARY_JOINT_MANUAL_CALIBRATE`, for M4LP1 / M4LT1 / M4LT2): lets the user manually move the joint to the desired zero position and trigger calibration. The execution sequence is:
  1. **ServoOff**: automatically powers off, allowing the user to freely move the joint to the target pose
  2. **SetHomePosition**: averages samples of the current position (~1 s / 50 frames) and writes them via `set_home_position()`
  3. **Target position reset**: after calibration, syncs the joint target to the measured position under the new coordinate frame, preventing a target jump shock the next time the task is enabled

  Difference from TID 4120 (AutoCalibrate): AutoCalibrate automatically drives the joint to find its boundary and then zeroes it; ManualCalibrate is triggered after manual positioning by the user, suitable for quick on-site calibration scenarios.

### 4.4.21

### 4.4.14

> 📅 2026-06-30 &nbsp;·&nbsp; Platform: `linux/arm64`

🐛 **Fixed**

- **Stand task enable shock (complete fix)**: pure `kp=0` damping is now applied across all three stages — `STAGE_INIT`, `STAGE_START`, and `STAGE_WARM_UP`:
  - `STAGE_INIT` (entered when switching from another task, 1 tick): added `output_joint_position = measured_position` with `kp=0`, avoiding a PD frame targeting zero position
  - `STAGE_START` (entry tick for any activation path): `kp=0`, target position follows the measured position
  - `STAGE_WARM_UP` (lasts 0.5 s): `kp=0`, target position continues to follow the measured position; after timeout, interpolation starts from the current measured position, gains are restored to normal, and the task enters `STAGE_PROCESS_01`
  - Added a stage-name prefix to log output for easier debugging
  - Also applies to `TaskM4LRotaryJointKneeRestrictionStand` (inherits the same `_function_meta`)

### 4.4.3

> 📅 2026-05-11 &nbsp;·&nbsp; Platform: `linux/arm64`

✨ **Added**

- **M4LT2 emergency stop high-damping protection task** (TID 4600): when the hardware emergency stop switch (ioboard) is triggered, joints no longer cut power immediately — the system automatically switches to high-damping mode (`kp=0`, `kd=80`) to prevent limbs from dropping suddenly. See [High-Damping Protection Task](/fourier-grx-M4/docs/en/tasks/emergency_stop_high_damping)
- **Configurable assist trigger force interface for Mark Time Assist (DT mode)**: new `assist_trigger_force_upper` / `assist_trigger_force_lower` parameters (defaults: `2.0 Nm` / `1.0 Nm`), configurable via panel `param_4` / `param_5`. Applies to TID 4119 and TID 4306

🔧 **Changed**

- Assist trigger force thresholds changed to absolute torque values: formula updated from `G[i] + offset` to `G[i] * 0.0 + offset` (gravity component zeroed). Experimental testing confirmed that including `G[i]` caused system instability

### 4.4.0

> 📅 2026-04-03 &nbsp;·&nbsp; Platform: `linux/arm64`

✨ **Added**

- **M4LP1** hardware model support (new variant with prismatic joints)
  - Automatic prismatic joint calibration (detects travel limits and resets to zero)
  - Task for moving prismatic joints to a specified length
  - Rotary joint calibration updated to a "travel to boundary then return" approach for improved reliability
- **M4LT2** hardware model support (FSA v3 async protocol); shares all algorithms and tasks with M4LT1
- **Knee Restriction** task set for limited knee range-of-motion scenarios:
  - Standing posture control (TID 4300)
  - Passive forward walking (TID 4301) · Passive marching in place (TID 4302)
  - Assisted forward walking — PD (TID 4303) · Assisted marching in place — PD (TID 4304)
  - Assisted forward walking — DT (TID 4305) · Assisted marching in place — DT (TID 4306)
- **Assisted mode subdivision**: split into separate PD and DT variants:
  - Assisted forward walking — PD (TID 4116) · Assisted marching in place — PD (TID 4117)
  - Assisted forward walking — DT (TID 4118) · Assisted marching in place — DT (TID 4119)
- **Planner** tasks: trajectory planning in a device-disabled state for upper-level display (TID 4401–4406)
- Automatic rotary joint calibration task (TID 4120): boundary detection → power-off drop → zero-point write
- Keyboard shortcut input support
- IO board communication and read/write support

🐛 **Fixed**

- Fixed multiple issues in prismatic joint length calibration
- Fixed flag set/clear inconsistencies
- Fixed `virtual_panel` not responding
- Fixed several issues during walking

🔧 **Changed**

- Tuned PD parameters for assisted mode (DT algorithm) to improve response stiffness
- Reduced false triggers in the load/energy detection algorithm
- Improved gait generator stability in knee-restriction scenarios
- Packaging now supports PyInstaller bundling of the `fi_fsa_v3` library (required for M4LT2)
