# 通信接口

* TOC
{:toc}

本页介绍 Fourier-GRX-M4 SDK 中 User 接口的通信机制，包括底层协议、自动发现流程、数据模型与配置方法。

---

## 架构概览

通信采用 **服务端 / 客户端** 模式：

```
┌───────────────────────────────────┐       局域网 (UDP)       ┌───────────────────────────────────┐
│           机器人主控板              │  ──── 状态数据 ────►     │         用户计算机（SDK）            │
│        (SyncServer)               │  ◄─── 控制指令 ────      │        (SyncClientSocket)          │
│                                   │                         │                                   │
│  • 广播自身 IP:Port（端口 9527）    │                         │  • 监听广播，自动发现服务端            │
│  • 发布机器人状态数据（50 Hz）      │                         │  • 接收状态，发送指令                 │
└───────────────────────────────────┘                         └───────────────────────────────────┘
```

- **服务端**：`fourier-grx` 运行于机器人主控板，启动 `SyncServerSocket`，持续向已知客户端推送机器人状态，并接收客户端发来的控制指令。
- **客户端**：在用户计算机（或同局域网的任意主机）上通过 Fourier-GRX-M4 SDK 实例化 `SyncClientSocket`，完成双向数据交换。

## 传输层

| 属性         | 值                                |
|------------|----------------------------------|
| 网络协议       | UDP（无连接，低延迟）                     |
| 序列化格式      | [msgpack](https://msgpack.org/)  |
| 数据端口       | `5566`（服务端监听）                    |
| 自动发现端口     | `9527`（广播端口）                     |
| 最大包大小      | 65 507 字节（UDP 理论上限）              |
| 默认通信周期     | 20 ms（50 Hz），可通过配置文件调整           |

> ℹ️ **说明**：UDP 不保证数据包的送达顺序或可靠性。在带宽充足的有线局域网中，丢包率通常可忽略不计。

## 自动发现流程

服务端启动后，会在后台线程中以约 **1 秒/次** 的频率向局域网广播地址（`255.255.255.255:9527`）发送包含自身 `IP` 和 `Port` 的 msgpack 数据包。

客户端在绑定本地随机端口后，启动后台线程监听广播端口 `9527`，收到第一个广播包后即完成发现，随后所有数据均通过数据端口 `5566` 收发。

```
服务端                          客户端
  │                               │
  │── 广播 {host, port} ─────────►│  (每 1 秒，端口 9527)
  │                               │  发现服务端
  │◄─────── 控制指令 (UDP 5566) ───│
  │──────── 状态数据 (UDP 5566) ──►│
  │                               │
```

如果机器人 IP 已知且固定，也可在实例化客户端时直接指定 `server_host` 和 `server_port`，跳过自动发现步骤：

```python
from fourier_grx.process.sync.fi_sync_client_socket import SyncClientSocket

client = SyncClientSocket(
    server_host="192.168.1.100",
    server_port=5566,
    auto_discover=False,
)
```

## 主题（Topic）与数据方向

通信内容按主题键（topic key）区分，共 6 个主题：

| 主题键    | 方向                | 说明                         |
|--------|-------------------|----------------------------|
| `comm` | 双向                | 通信状态与心跳信息                  |
| `core` | 服务端 → 客户端         | 底层核心状态（关节传感器原始数据等）         |
| `robot` | 服务端 → 客户端        | 机器人运动状态（关节位置/速度/力矩等）       |
| `task` | 双向                | 任务状态与任务控制指令                |
| `grx`  | 双向                | fourier-grx 通用机器人（主要为人形）数据  |
| `rehab` | 双向               | 康复机器人专用数据                  |

**数据方向说明**：
- **服务端 → 客户端**：机器人当前状态，通过 `read_to_dict()` / `read_from_dict()` 序列化/反序列化。
- **客户端 → 服务端**：控制指令，通过 `write_to_dict()` / `write_from_dict()` 序列化/反序列化。

各主题下的具体字段请参见 [User 接口参考](/fourier-grx-M4/docs/reference/user)。

## 配置文件参数

在 `fourier-grx` 配置文件（通常位于 `~/.fourier/fourier-grx/config/`）中，以下两个配置块控制通信行为：

```yaml
# 控制 SyncServer 是否启动及通信周期
communication:
  enable: true        # true = 启动通信服务端
  type: "socket"      # 通信后端类型，M4L 标准使用 "socket"
  period: 0.02        # 通信发布周期（秒），默认 0.02 s（50 Hz）

# 控制 SyncClient 子进程是否启动
sync:
  enable: true        # true = 启动内部同步子进程
  period: 0.02        # 子进程发布周期（秒）
```

> ⚠️ **注意**：`communication.enable` 和 `sync.enable` 需**同时**设置为 `true`，通信功能才能正常工作。仅在 **开发者模式** 下，两者才默认开启。

## 网络建议

| 建议              | 说明                        |
|-----------------|---------------------------|
| 使用有线以太网连接       | 最稳定，延迟最低                  |
| 避免同时使用有线和无线网卡   | 双网卡可能导致路由冲突，影响自动发现和数据传输   |
| 确保机器人与用户计算机在同一子网 | 广播包不跨越路由器，跨子网无法自动发现       |
| 关闭防火墙或开放相应端口    | UDP 5566（数据）、UDP 9527（广播） |
