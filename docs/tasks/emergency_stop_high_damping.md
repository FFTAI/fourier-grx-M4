# 高阻尼保护（急停响应）

## 任务信息

任务值 (TID): **4600**

适用机型：**M4LT2**

任务描述：

- 当机器人检测到 **硬件急停开关**（ioboard 急停按钮）被触发时，控制程序自动切换到该任务。
- 与直接断电（`TASK_SERVO_OFF`）不同，该任务保持关节上电状态，以 **高阻尼、零弹簧力** 的 PD 模式对旋转关节进行控制：
  - `kp = 0`：不施加复位弹簧力，关节可被外力被动活动；
  - `kd = 80（默认）`：施加速度阻尼制动力，消耗关节运动能量，防止肢体快速跌落或甩动；
  - 目标位置跟随当前测量位置，每帧实时更新，无回弹效果。
- 该模式可有效降低急停后机器人肢体无控失控的风险，为操作者提供更安全的保护时间窗口。

> ⚠️ **注意**：
>
> 此任务由控制程序在 `flag_emergent_stop` 被置位时**自动触发**，不建议通过手柄或 User 接口手动切换到该任务。
> 急停解除后，机器人将继续保持高阻尼状态，需手动切换其他任务（如 `TASK_STAND_CONTROL`）才能恢复正常运动控制。

任务参数：

该任务无可配置参数。PD 参数 `kd` 值需由硬件工程师在 `fi_algorithm_m4l_rotary_joint_high_damping.py` 中根据实际关节惯量调参。

## 触发条件

该任务由以下两种急停条件之一触发（两者均会设置 `flag_emergent_stop`）：

| 触发源 | 触发条件 | 说明 |
|--------|---------|------|
| ioboard 急停开关 | `ioboard.emergency_state == True` | 用户按下机器人背包上的硬件急停按钮 |
| 全执行器通信超时 | 所有执行器连续超时 ≥ 50 个控制周期 | 通信异常导致的保护性触发 |

## 模块信息

该任务暂无独立模块指令，不支持通过模块接口进行分关节控制。

## 接口信息

状态接口：

| 接口参数     | 接口映射关系                              | 说明                          |
|------------|----------------------------------------|-------------------------------|
| 急停标志     | `robot.flag_robot_emergent_stop`       | 高阻尼任务运行期间始终为 SET    |

指令接口：

该任务无需额外指令参数，由控制程序自动触发，无法通过 TID=4600 手动切换启动。

## 与执行器失能的区别

| 特性         | 高阻尼保护（4600）              | 执行器失能（36）               |
|------------|-------------------------------|-----------------------------|
| 触发方式     | 硬件急停开关自动触发（M4LT2）    | 手柄 logo 键 / User 接口手动触发 |
| 关节上电状态 | 保持上电，输出阻尼力矩            | 断电，无力矩输出                |
| 安全特性     | 软制动，肢体不会瞬间跌落          | 机器人立即瘫软                  |
| 恢复方式     | 手动切换其他任务                 | 需重新上电并切换任务             |
